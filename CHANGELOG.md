## v1.1 — 2026-04-12

基于得润电子(002055)和喜临门(603008)的实测反馈，修复盲区并优化流程。

### 新增规则
- **Rule 5.8 其他应收款异常**: 检查 `oth_receiv/total_assets` 比例 + 年报"非经营性占用"段落，覆盖关联方资金占用信号
- **Rule 5.9 监管处罚/立案调查**: 搜索年报全文中的"立案""警示函""处罚决定""监管措施"等关键词，在审计意见之外捕获独立预警
- 规则总数 28 → 30

### PDF 处理优化
- 使用 `pdftotext -layout` 将年报全文转为文本文件后一次性读入，替代之前逐页读取（每次限20页）的方式
- 解决了前五大客户/供应商、运费明细等关键信息被遗漏的问题

### 报告输出规范
- Phase 5 新增 MANDATORY 指令：完整报告必须先输出到终端，不得省略
- 新增"积极信号"段（低/中风险时输出）
- 新增 7 条严格输出格式规则（verdict 格式、detail 必须含数据、→ 解释等）
- Phase 6 明确执行顺序：先输出 → 再保存

### 文档
- README 更新规则数 28 → 30，Layer 5 条数 7 → 9，补充 Rule 5.8/5.9 说明和案例
- 新增 CLAUDE.md 项目上下文文件

## v1.0 — 2026-04-12

初始版本，基于唐朝《手把手教你读财报（新准则升级版）》构建完整排雷体系。

### 规则体系
- 7 层 28 条规则（Layer 0-6），覆盖利润表、现金流、资产负债表、交叉验证、非财务信号、行业风险
- 评分体系：分层权重 + 组合加分 + 四级风险等级
- Layer 0 门槛检查（审计意见 + 年报按时披露）一票否决

### 数据源
- Tushare API（主）：结构化财务数据，10+ 年历史
- 年报 PDF（辅）：附注、董事会报告、关联交易等定性分析
- 复用龟龟投资框架的 `tushare_collector.py`

### 知识库
- 8 章读书笔记（`手把手读财报/ch1-ch8`）
- 原书 PDF

### Skill
- `/minesweeper <股票代码> [年份]` 一键排雷
- 自动获取数据 → 逐条检查 → 输出风险等级报告

### 文档
- README.md：28 条规则逐条标注原书依据与排雷逻辑
